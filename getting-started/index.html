Getting started

This guide will take you through the Cake pipeline. This is only one of many ways to install Cake depending on your needs.

1. Install Cake

The absolutely easiest way of getting started with Cake, is to download a Powershell script that will:

  • Install Cake.exe
  • Restore NuGet dependencies for tools used by the build script.
  • Run the build script.

This powershell script can be used locally as well as on a build server.

  • Powershell
  • Command line
  • Manual
PS> "build.ps1","build.cake"|%{Invoke-RestMethod -Uri "https://raw.githubusercontent.com/cake-build/bootstrapper/master/res/scripts/$($_)" -OutFile $_}
CMD> PowerShell.exe -NoLogo -NonInteractive -NoProfile -ExecutionPolicy RemoteSigned -Command "'build.ps1','build.cake'|%{Invoke-RestMethod -Uri "https://raw.githubusercontent.com/cake-build/bootstrapper/master/res/scripts/$($_)" -OutFile $_};"

Download the following files and save them in the project folder.

  • build.ps1 (bootstrapper script)
  • build.cake (build script)

2. Take a look at the build script

The build script (build.cake) that we downloaded is extremely generic, but a very good starting point for trying out Cake.

3. Execute the build script

To execute the build script, simply call ./build.ps1 from Powershell.

You will see the build script do the following things:

  1. Clean the solution output folders.
  2. Restore all NuGet packages.
  3. Build all solutions.

This is an extremly generic build script, but a good starting point to get up and running.
